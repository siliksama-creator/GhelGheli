# Mobile Flutter App

```bash
flutter pub get
flutter create --platforms=android .
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:4000
```

## ساختار UI (بازطراحی ۲۰۲۶)

کد رابط کاربری قبلاً همه در یک فایل `main.dart` سنگین (بیش از ۱٬۵۰۰ خط) بود که
باعث می‌شد استایل‌دهی حرفه‌ای و نگهداری آن سخت شود. حالا `lib/main.dart` فقط
نقطه‌ی شروع اپ (تم، لوکال، مسیر بین صفحه ورود/کاربر/مدیر) است و بقیه‌ی کد به
شکل ماژولار در این پوشه‌ها سازمان‌دهی شده:

```
lib/
  theme/        سیستم طراحی: رنگ‌ها، تایپوگرافی (فونت وزیرمتن)، توکن‌های
                فاصله‌گذاری (شبکه ۸dp)، گردی گوشه‌ها، انیمیشن، و
                ThemeExtension برند برای گرادیان‌ها/سایه‌های سفارشی
                (روشن + تیره، Material 3)
  core/         توابع و ثابت‌های عمومی بدون وابستگی به UI (آواتارها، پارس عدد)
  widgets/      کامپوننت‌های عمومی قابل استفاده در کل اپ:
                AppCard، GradientPanel، SectionHeader، EmptyState/ErrorBanner،
                AvatarImage، FeaturePill/StatusBadge، AsyncSection، HeroLogo
  screens/
    auth/       صفحه ورود/ثبت‌نام/ورود مدیر + اسپلش
    shared/     ویجت‌های مشترک بین کاربر/مدیر (HeroHeader، FootballCard،
                RankTile، شیت پروفایل عمومی)
    user/       ۶ تب اپ کاربر (خانه، جوایز، لیگ، چت، پشتیبانی، پروفایل)
    admin/      ۱۰ بخش مدیریت + ویجت‌های فرم مشترک (FormSection، StatCard،
                ImageUrlField) و شل واکنش‌گرا (Drawer در موبایل،
                Side-rail دائمی در تبلت/دسکتاپ)
```

نکات مهم بازطراحی:

- **هیچ منطق و endpoint‌ای تغییر نکرده** — همه‌ی API calls، مسیرها، مقادیر
  پیش‌فرض و رفتار دقیقاً همان چیزی است که در نسخه‌ی قبلی وجود داشت.
- فونت پیش‌فرض از Tahoma به **وزیرمتن** (Vazirmatn) تغییر کرده که خوانایی و
  تنوع وزن بسیار بهتری برای فارسی/RTL دارد (`assets/fonts/`).
- تم روشن و تیره کامل با Material 3 پیاده‌سازی شده (`theme/app_theme.dart`)
  و از طریق دکمه بالای صفحه در هر دو اپ کاربر و مدیر قابل تعویض است.
- شل مدیریت اکنون واکنش‌گرا است: در صفحه‌های بزرگ (تبلت/دسکتاپ) نوار کناری
  دائمی نمایش داده می‌شود و در موبایل به Drawer برمی‌گردد.
- کارت‌ها، دکمه‌ها، ورودی‌ها، اسنک‌بار، دیالوگ و باتم‌شیت‌ها یک ظاهر و رفتار
  یکپارچه (radius، سایه، فاصله‌گذاری روی شبکه ۸dp) دارند.
- `flutter analyze` بدون هیچ خطا یا هشداری اجرا می‌شود.

## APK از GitHub
Workflow `Build Android APK` بعد از push اجرا می‌شود و APK دیباگ نصب‌شدنی را در بخش Artifacts قرار می‌دهد.

## Firebase / FCM
برای Push Notification واقعی، فایل `android/app/google-services.json` و تنظیمات FlutterFire پروژه Firebase باید قبل از build production اضافه شود. در debug بدون این فایل، اپ با try/catch اجرا می‌شود ولی دریافت push فعال نیست.

## ورود تست بدون پیامک

بعد از اجرای `npm run seed:admin` در backend، در اپ از حالت «ورود مدیر» با نام کاربری Admin و رمز Ali@0142 استفاده کنید.

## حالت مدیریت داخل اپ

اپ Flutter حالا دو نوع ورود دارد:

- ورود کاربر عادی با موبایل/رمز عبور → بخش عادی اپ
- ورود مدیر با username/password از مسیر امن `/api/admin/auth/login` → بخش مدیریت داخل اپ

اطلاعات نقش مدیر داخل APK تصمیم‌گیری نمی‌شود؛ backend توکن JWT ادمین با `type=admin` صادر می‌کند و همه endpointهای مدیریتی همچنان با `adminAuth` و role کنترل می‌شوند.

بخش مدیریت داخل اپ شامل داشبورد، مدیریت کارت و کد، آپلود عکس کارت از گالری، ثبت تکی/دسته‌جمعی کد، جوایز، لیگ، کاربران، چت، پشتیبانی، اطلاعیه‌ها، ادمین‌ها و audit log است.

## محدودیت چت از داخل اپ مدیریت

در Admin Mode مسیر «تنظیمات» اضافه شده است. مدیر می‌تواند حداقل امتیاز تاریخی لازم برای ورود/ارسال پیام در چت را تعیین کند. این مقدار سمت backend ذخیره و در Socket.io و API چت enforce می‌شود.

برای مشاهده سریع محیط مدیریت در APK تست، حالت «ورود مدیر» با نام کاربری Admin و رمز Ali@0142 در صفحه ورود اضافه شده است.

## جوایز و نوار پیشرفت

مدیر در حالت مدیریت داخل اپ می‌تواند تا ۳۰ جایزه تعریف کند، برای هر جایزه نام، عکس، امتیاز مورد نیاز، نوع و توضیح جایزه ثبت کند. نوار پیشرفت صفحه خانه کاربر، جایزه بعدی را از همین سطح‌های تعریف‌شده محاسبه می‌کند.

## ورود تست به‌روز شده

برای تست، در صفحه ورود حالت «ورود مدیر» را انتخاب کنید:

```text
Username: Admin
Password: Ali@0142
```

رمز به‌صورت دکمه ورود یک‌کلیکی داخل اپ hardcode نشده است؛ باید در فیلد رمز وارد شود.
